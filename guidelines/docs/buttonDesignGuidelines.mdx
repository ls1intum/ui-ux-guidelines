# Test

These guidelines define how buttons should look, behave, and be implemented across all products.
The goal is to ensure a **consistent, accessible, and intuitive** user experience while maintaining **flexibility and
scalability**.

## ‚úÖ TL;DR

- Use the correct variant for every button
- Only **one primary button per page**
- Destructive buttons always open a confirmation
- Use spacing and corner radius consistently
- Icon buttons must be circular and have labels
- Don‚Äôt use links for actions or buttons for navigation
- Avoid visual overload ‚Äì use menus for secondary actions

## üß© Button Variants

Use **semantic variants** based on the button‚Äôs purpose, not its appearance.

Semantic variants define the function of a button in the context of the UI, such as the ones below. Not all the
following variants are needed in all applications.
